Design System¶

The AtiFLOW MTS Tablet design system is built on top of Material UI (MUI) v5 with a custom theme defined in src/theme.ts. All design tokens are applied globally via ThemeProvider in App.tsx.

Typography¶

Font family: "Inter", "Roboto", "Helvetica", "Arial", sans-serif

The font is loaded via Google Fonts (configured in main.tsx/index.html). Inter is chosen for its industrial clarity and excellent legibility at small sizes on screens.

Type Scale (MUI overrides)¶

Variant Weight Size Usage
h5 700 MUI default (~1.5rem) Large section headings
h6 600 MUI default (~1.25rem) Page titles, dialog titles
subtitle1 500 MUI default Section sub-headings
subtitle2 600 MUI default Card headings
body2 400 0.8125rem (13px) Secondary body text, helper text
caption 400 0.75rem (12px) Labels, metadata, timestamps

Custom Typography in Components¶

Many components use inline sx typography because they deviate slightly from the MUI scale for pixel-perfect density:

Usage fontSize fontWeight
Sub-header page titles 1rem 700
Card headings 0.875rem–0.9375rem 600–700
Item names in lists 0.875rem 600
Monospace codes inherited + fontFamily: 'monospace' 400
Micro labels / status 0.7rem 500–600
App bar brand name 0.8rem 700
App bar station name 0.65rem 400

Spacing System¶

MUI's default 8px grid is used throughout. Common spacing values:

MUI unit px Usage
0.5 4px Tight chip/icon gaps
1 8px Compact gaps within components
1.5 12px Standard gap between form fields
2 16px Card padding (compact), section gaps
2.5 20px Standard card padding (px), content area padding
3 24px Dialog padding, large card padding
3.5 28px Login card inner padding
4 32px Success screen padding

MUI token px Usage
shape.borderRadius 6 Global default (all MUI components inherit)
borderRadius: 1 4px Inline steppers, small chips
borderRadius: 1.5 6px (= base) Standard cards, buttons, outlined inputs

borderRadius: 4 16px Chip global override
MuiDialog.paper 10 Dialog containers

Touch Targets¶

All interactive elements must meet the 44px minimum touch target requirement for industrial gloved-hand usage. This is enforced via MUI theme overrides:

MuiButton: {
  styleOverrides: {
    root: { minHeight: 44 },
  },
},
MuiTab: {
  styleOverrides: {
    root: { minHeight: 44 },
  },
},

Note: Inline IconButton components within dense list rows use width: 36, height: 36 — these are in tight-fit contexts where larger targets would crowd the layout. Full-width action buttons always meet 44px.

Elevation & Shadow¶

MUI's elevation system is used sparingly:

Component elevation Notes
AppBar 0 Custom border: 1px solid rgba(255,255,255,0.08)
BottomNav Paper 0 Custom border: borderTop: '1px solid divider'
Login Paper 0 Glassmorphism: bgcolor: rgba(255,255,255,0.05), backdropFilter: blur(8px)
All cards variant="outlined" No shadow by default
Primary action card hover — boxShadow: '0 8px 32px rgba(0,150,136,0.12)'
Button containedPrimary 0 boxShadow: none overridden to remove default MUI elevation

Dividers¶

Divider components use the theme default colour #E8ECEF. Sub-header bars use borderBottom: '1px solid', borderColor: 'divider' consistently.

Dark Mode¶

The application does not support dark mode. The login page uses a dark background (TOPBAR_BG = '#263238') with a semi-transparent glass card, but this is not a theme toggle — it is the fixed login screen style.
