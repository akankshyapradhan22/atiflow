Return Trolley Page¶

Route: /requester/return-trolley Component: src/pages/requester/ReturnTrolleyPage.tsx Auth: Protected Entered from: Create Request (Return Trolley zone) Accent colour: Purple #6A1B9A throughout

Purpose¶

Allows operators to schedule a pickup of an empty container (typically a trolley) and return it to storage. The operator identifies the container either by scanning its QR code or entering its details manually.

handleManualLookup() Logic¶

const handleManualLookup = () => {
  const key = containerId.trim().toUpperCase();
  const found = MOCK_CONTAINERS[key];

  if (found) {
    setResolved(found);           // exact match â€” real data
    setNotFound(false);
  } else {
    // Try to construct from form fields
    const activeContainer = mockContainers[containerTypeIdx];
    const subtype = activeContainer.subtypes.find((s) => s.id === subtypeId);
    if (subtypeId && subtype) {
      setResolved({
        containerId: containerId || `${TYPE_LABELS[activeContainer.type].charAt(0)}-NEW`,
        containerType: TYPE_LABELS[activeContainer.type],
        subtypeName: subtype.name,
        location: 'Unknown â€“ manual entry',
        status: 'Empty',
        lastUsed: 'â€”',
      });
      setNotFound(false);
    } else {
      setNotFound(true);          // Neither ID match nor form complete
    }
  }
};

Three outcomes: 1. ID found in MOCK_CONTAINERS â†’ real details shown 2. ID not found but type+sub-type selected â†’ synthetic details shown ("Unknown â€“ manual entry" location) 3. ID not found and no sub-type â†’ notFound = true â†’ error helperText shown, resolved stays null
