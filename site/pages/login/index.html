Login Page¶

Route: /login Component: src/pages/login/LoginPage.tsx Auth: Public (unauthenticated access only â€” redirects to /requester/create if already logged in)

Purpose¶

The login page authenticates the device operator before they can access the MTS tablet interface. It is the first screen presented on a fresh session or after logout. The dark background and station info banner visually reinforce that this is a secure, station-bound device.

Enter Key Support¶

Both text fields have onKeyDown={handleKey} which triggers handleLogin() when e.key === 'Enter'. This supports hardware keyboard entry (barcode scanner, USB keyboard).

Already Logged In¶

<Route path="/login" element={user ? <Navigate to="/requester/create" replace /> : <LoginPage />} />

If the user navigates to /login while already authenticated, they are immediately redirected. No flash of the login screen.
