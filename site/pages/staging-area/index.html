Route: /requester/staging Component: src/pages/requester/StagingAreaPage.tsx Auth: Protected Bottom Nav: "Staging Area" tab (index 2)

Purpose¶

Provides a visual and tabular view of all staging areas assigned to this device. Operators can: - See which staging cells are occupied, reserved, or empty - Switch between a visual grid layout and a detailed table list - Enter Manage mode to edit individual cell metadata (container ID, type, ageing, notes)

UX Logic¶

Edits are local only¶

The edits Record is never written to context or API. Changes are session-scoped â€” navigating away and back resets the staging area to the original mock data. In production, saves would trigger a PATCH to the MTS backend.

Status override¶

const getStatus = (cell: StagingCell): CellStatus =>
  (edits[cell.id]?.status ?? cell.status) as CellStatus;

The edit record takes precedence over the original cell status for display in both grid and list views and for statistics calculation.
