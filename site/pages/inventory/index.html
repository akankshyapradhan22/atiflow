UI Components¶

Header¶

  • Title "Live Inventory" (fontWeight: 700)
  • Caption "Real-time material availability for your workflow"

Summary Chips¶

4 aggregate chips computed from filtered rows (update in real time as search filters):

Chip Value Colour
Available Sum of all row.available Teal
Reserved Sum of all row.reserved Amber
In Transit Sum of all row.inTransit Blue
Pre-Processing Sum of all row.preProcessing Grey

These totals change as the user types in the search box.

Search Bar¶

  • OutlinedInput, max-width 400px, white background
  • Filters by row.sku and row.subSkuType (case-insensitive)

Table¶

MUI Table with size="small" and stickyHeader inside a flex-1 TableContainer with overflow: 'auto'.

Columns:

Column key Label Alignment Colour
sku SKU left default
subSkuType Sub-SKU Type left default
produced Produced center default
preProcessing Pre-Proc. center default
available Available center Teal
reserved Reserved center Amber #F57C00
inTransit In Transit center Blue #1565C0
consumed Consumed center default
total Total center default

Column header colours are applied via the color property in the COLUMNS definition.

Cell Rendering¶

Available column: 

<Typography sx={{ fontWeight: 700, color: row.available > 0 ? PRIMARY : 'text.disabled', fontSize: '0.8125rem' }}>
  {row.available}
</Typography>
- > 0: Bold teal number - === 0: Disabled grey

Reserved column: 

{row.reserved > 0 ? (
  <Chip label={row.reserved} size="small" sx={{ bgcolor: 'rgba(245,124,0,0.1)', color: '#F57C00' }} />
) : <Typography sx={{ color: 'text.disabled' }}>0</Typography>}

In Transit column: Same pattern as Reserved but with blue chip.

Pre-Processing column: 

{row.preProcessing > 0 ? (
  <Chip label={row.preProcessing} size="small" sx={{ bgcolor: '#F5F5F5' }} />
) : row.preProcessing}  // shows 0 as plain text

No inventory data found

Table Container Layout¶

The table is inside a Paper variant="outlined" which uses height: '100%' and display: 'flex', flexDirection: 'column'. The TableContainer has flex: 1, minHeight: 0, overflow: 'auto' â€” this is the key that makes the sticky header work within the constrained viewport:

<Box sx={{ flex: 1, minHeight: 0, overflow: 'hidden', px: 2.5, pb: 2 }}>
  <Paper sx={{ height: '100%', display: 'flex', flexDirection: 'column', ... }}>
    <TableContainer sx={{ flex: 1, minHeight: 0, overflow: 'auto' }}>
      <Table stickyHeader>
        ...
      </Table>
    </TableContainer>
  </Paper>
</Box>

The sticky header pins the column labels while scrolling through rows.

UX Logic¶

Totals calculation¶

const totals = filtered.reduce(
  (acc, row) => ({
    produced: acc.produced + row.produced,
    // ... all columns
  }),
  { produced: 0, preProcessing: 0, available: 0, reserved: 0, inTransit: 0, consumed: 0, total: 0 }
);

The summary chips show totals for the filtered dataset, so searching for "Widget" shows only Widget A inventory in the chips.

Edge Cases¶

Scenario Behaviour
Search: no match Empty state message shown; summary chips show all zeros
All items available Available chip shows total; no grey "0" values
Pre-processing > 0 Shown as grey chip (not teal) to distinguish from available
Long SKU name Text wraps (no truncation in table cells)
