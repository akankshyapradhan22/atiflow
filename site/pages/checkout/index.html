UX Logic¶

Inline Container Add vs. ContainerSelectionPage¶

Two paths exist to add containers: 1. CheckoutPage â†’ "Add Containers" (inline, local state) 2. CreateRequestPage â†’ not shown â€” containers have their own dedicated route via ContainerSelectionPage â†’ ContainerCheckoutPage

The inline approach is for operators who realise mid-order they need containers. It uses local state rather than containerCart context.

handleAddContainer()¶

const handleAddContainer = () => {
  if (!selSubtypeId) return;
  const subtype = activeContainer.subtypes.find((s) => s.id === selSubtypeId);
  if (!subtype) return;
  setAddedContainers((prev) => {
    const existing = prev.findIndex((c) => c.subtypeId === selSubtypeId);
    if (existing >= 0) {
      return prev.map((c, i) => i === existing ? { ...c, qty: c.qty + containerQty } : c);
    }
    return [...prev, { containerId, containerType, subtypeId, subtypeName, qty: containerQty }];
  });
  setSelSubtypeId('');    // reset picker
  setContainerQty(1);     // reset qty
};
